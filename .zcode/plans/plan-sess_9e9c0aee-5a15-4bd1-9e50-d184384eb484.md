## Plan corregido v2: Optimización para audios 10-30s (fundamentado en verificación de API)

### Por qué este plan reemplaza al anterior

La auditoría técnica del plan previo destapó **6 errores de ingeniería graves** que habrían producido cero mejora o comportamiento incorrecto:

| # | Error del plan anterior | Verificado contra |
|---|---|---|
| 1 | "`single_segment=false` divide en ventanas 30s con solape" para audio 10-30s | **FALSO**: el windowing SOLO aplica para audio >30s. Para <30s el loop itera una vez sin importar el flag. Cero impacto en latencia. (`whisper.cpp:7001-7763`) |
| 2 | Fórmula `audio_ctx = audio_secs * 150` | **MAL por 3×**: `audio_ctx` son mel-frames reducidos por 2, donde 1500=30s=**50 frames/s**. Fórmula correcta: `audio_secs * 50`. (`whisper.h`, `WHISPER_HOP_LENGTH=160`) |
| 3 | "`max_len(60)` son tokens" | **FALSO**: son **caracteres**. 60 chars fragmenta palabras. El plan subía a 120, también mal planteado. (`whisper.h:508`) |
| 4 | Añadir crate `voice-activity-detection` externo | **DUPLICACIÓN**: whisper-rs 0.16 YA trae VAD nativo (`enable_vad`+`WhisperVadParams`) que **recorta audio físicamente antes del encoder**. (`whisper.cpp:7749-7763`) |
| 5 | `BufferState` con `Vec<AudioSegment>` + 1 SttJob por segmento | **ROMPE INYECCIÓN**: segmentos desordenados con 2 workers, múltiples Ctrl+V, parpadeo de estado. |
| 6 | "Lock contention fix" adquiere el lock dos veces | **EMPEORA** el original. Además, el lock actual (~µs por memcpy de 2KB) no es un cuello real. |

### Decisiones confirmadas contigo
- **VAD**: nativo de whisper-rs (`enable_vad`), no crate externo.
- **Segmentos**: VAD solo recorta, 1 texto por hold (sin fragmentar inyección).
- **Warm-up**: ampliar a 30s para cubrir rutas multi-segmento del decoder (recomendado para tu caso 10-30s).
- **WhisperState**: explorar pool reutilizable (ahorra allocs).

### FASE 1 — Corrección de bugs existentes (impacto: correctitud, riesgo: bajo)

**1.A — Corregir fórmula `audio_ctx`** (`whisper_cpp.rs:172-178`)
```rust
// ANTES (mal, 150 frames/s): ctx_frames = ((audio_secs + 1.0) * 150.0)
// DESPUÉS (correcto, 50 frames/s, con padding de 2s y techo 1500):
let ctx_frames = (((audio_secs + 2.0) * 50.0) as i32).clamp(100, 1500);
params.set_audio_ctx(ctx_frames);
```
El padding de +2s evita truncar el último segundo por redondeo. El piso de 100 (2s) evita ctx demasiado pequeño.

**1.B — Corregir `max_len`** (`whisper_cpp.rs:157`)
- `max_len` mide **caracteres**, no tokens. `set_max_len(60)` fragmenta palabras (60 chars ≈ 10-12 palabras).
- **Cambio**: **eliminar `set_max_len`** por completo (default=0=sin límite). La anti-alucinación real ya está cubierta por `entropy_thold(2.4)` + `logprob_thold(-1.0)` + `suppress_blank` + `suppress_nst`, que son los mecanismos correctos. Si se quiere segmentación visual, activar `set_split_on_word(true)` con un `max_len` alto (200+), pero para dictado hold-to-talk el default sin límite es correcto.

**1.C — Corregir comentario engañoso de `single_segment`** (`whisper_cpp.rs:159-164`)
- El comentario actual afirma que ahorra coste de encoder para audio corto. **Es inexacto**: `single_segment` solo afecta el decoder/segmentación, no el encoder.
- **Cambio**: mantener `set_single_segment(true)` (correcto para hold-to-talk: produce salida limpia sin marcas de timestamp intermedias), pero **reescribir el comentario** explicando que suprime la división de timestamps dentro de una ventana, no que ahorra encoder.
- **No hacerlo condicional**: no aporta nada para audio <30s.

### FASE 2 — VAD nativo de whisper-rs (impacto: alto, riesgo: medio)

El VAD nativo **recorta físicamente el audio antes del encoder** (verificado `whisper.cpp:7749-7763`): concatena solo segmentos con voz separados por 0.1s de silencio sintético, y remapea timestamps automáticamente. Para un dictado de 20s con 5s de pausas, el encoder ve ~15s en lugar de 20s → ~25% menos cómputo.

**2.A — Descarga del modelo VAD GGML** (`main.rs` boot)
- Archivo: `ggml-silero-v5.1.2.bin` (~2MB) desde `https://huggingface.co/ggml-org/whisper-vad`.
- Formato **GGML obligatorio** (NO ONNX — whisper.cpp exige `GGML_FILE_MAGIC`).
- Descargar al mismo `models_dir` que el modelo whisper, si no existe.
- Añadir helper `resolve_vad_model_path()` paralelo a `resolve_models_dir()`.

**2.B — Configurar VAD en `transcribe()`** (`whisper_cpp.rs`, nuevo campo `vad_model_path: Option<PathBuf>`)
```rust
// En WhisperCpp struct: añadir campo
vad_model_path: Option<PathBuf>,

// En transcribe(), antes de state.full():
if let Some(vad_path) = &self.vad_model_path {
    let vad_path_str = vad_path.to_str().ok_or_else(...)?;
    params.set_vad_model_path(Some(vad_path_str));  // OBLIGATORIO antes de enable_vad
    let mut vad_params = WhisperVadParams::new();   // defaults razonables
    vad_params.set_threshold(0.5);
    vad_params.set_min_speech_duration_ms(250);
    vad_params.set_min_silence_duration_ms(500);     // 500ms para recortar pausas largas
    vad_params.set_speech_pad_ms(30);
    params.set_vad_params(vad_params);
    params.enable_vad(true);                         // paniquea si path no seteado
}
```
- **Orden crítico**: `set_vad_model_path` → `set_vad_params` → `enable_vad(true)`. Invertir paniquea (`"Set a VAD model path before calling enable_vad"`).
- `min_silence_duration_ms(500)`: pausas >500ms se recortan. Tú decides — si dictas fluido, bajar a 300ms; si haces pausas largas entre frases, subir a 700ms.
- Si `vad_model_path` es `None` (no se descargó), el código no activa VAD y todo funciona como antes — **fallback graceful**.

**2.C — Cableado en `main.rs`**
- Resolver `vad_model_path` al boot, pasarlo al `WhisperCpp` builder:
```rust
let vad_path = resolve_vad_model_path(&models_dir);  // Option<PathBuf>
let mut stt = WhisperCpp::with_language(&snap.language_ui)
    .with_runtime(gpu_config, n_threads_per_worker)
    .with_vad(vad_path);  // nuevo builder method
```

### FASE 3 — `n_threads` dividido entre workers + pool de WhisperState (impacto: medio, riesgo: medio)

**3.A — `n_threads_per_worker`** (`main.rs:120-125` + `whisper_cpp.rs:108-113`)
- Hoy: 2 workers × `min(cores,8)` threads = oversubscription en CPUs de 8 cores.
- **Cambio**: `total = min(cores, 8)`, `per_worker = max(2, total / STT_WORKERS)`.
- En CPU de 8 cores: cada worker usa 4 threads (8 total, sin oversubscription).
- En CPU de 16 cores: cada worker usa 8 threads (16 total, óptimo).

**3.B — Pool de `WhisperState` reutilizable** (`whisper_cpp.rs`)
- Verificado: `WhisperState` es `Send + Sync` en 0.16 (el comentario actual en líneas 10-11 está desactualizado).
- En lugar de `ctx.create_state()` por cada `transcribe()`, mantener un pool thread-local:
```rust
// Por worker thread, vía thread_local!
thread_local! {
    static WHISPER_STATE: RefCell<Option<WhisperState>> = RefCell::new(None);
}
// En transcribe():
let state = WHISPER_STATE.with(|s| {
    let mut s = s.borrow_mut();
    s.get_or_insert_with(|| ctx.create_state().expect("create_state"))
});
```
- **Cuidado**: `WhisperState` mantiene buffers ggml entre llamadas. Reusarlo significa que la segunda inferencia puede tener estado residual del contexto previo. Hay que llamar `state.full_n_segments()` y limpiar, o aceptar que `no_context=true` + `single_segment=true` ya aíslan cada llamada. **Mitigación**: mantener `no_context=true` (ya activo) que previene contaminación cross-llamada.
- **Alternativa conservadora si el pool complica**: dejar `create_state()` por llamada (alloc pequeña vs inferencia). El pool es optimización secundaria.

### FASE 4 — Warm-up ampliado + métricas (impacto: bajo-medio, riesgo: bajo)

**4.A — Warm-up de 30s** (`whisper_cpp.rs:235-245`)
```rust
// ANTES: vec![0.0; 16_000]  (1s)
// DESPUÉS: vec![0.0; 16_000 * 30]  (30s)
let silence: Vec<f32> = vec![0.0; 16_000 * 30];
```
- Cubre rutas de decoder multi-segmento y calienta el VAD si está activo.
- Sube el tiempo de arranque ~5-10s. Loguear duración (ya existe `main.rs:155-162`).

**4.B — `print_timings()` tras cada inferencia** (`whisper_cpp.rs`, tras `state.full()`)
- whisper-rs 0.16 expone `ctx.print_timings()` (a stderr). **No expone los campos programáticamente** (`whisper_get_timings` no está wrapped).
- Activar condicionalmente con env `OIDO_LOG_TIMINGS=1`:
```rust
if std::env::var("OIDO_LOG_TIMINGS").as_deref() == Ok("1") {
    ctx.print_timings();  // stderr: load_ms, sample_ms, encode_ms, decode_ms, batchd_ms, prompt_ms
}
```
- Permite auditar dónde se va el tiempo REAL sin añadir dependencias.

### FASE 5 — Pulido menor (impacto: bajo, riesgo: bajo)

**5.A — `max_pending` resampler** (`capture.rs:298`): `*64 → *128`. Reduce drops en picos de CPU.

**5.B — `STT_QUEUE_CAP: 4 → 6`** (`pipeline.rs:92`): margen extra sin riesgo de OOM.

**5.C — NO tocar el lock de `BufferState`**: la auditoría confirma que la contención es trivial (~µs por memcpy de 2KB). El "fix" del plan anterior empeoraba el código. Dejar como está.

### Estimación de mejora (realista, basada en verificación)
- **Audio 15s con 4s de pausas**: VAD recorta a ~11s → encoder -27% → ~6s → ~4.5s total (-25%).
- **Audio 25s con 6s de pausas**: VAD recorta a ~19s → encoder -24% → ~9s → ~7s total (-22%).
- **Oversubscription eliminado** (n_threads/worker): -10-15% en CPU de 8 cores con 2 workers concurrentes.
- **audio_ctx corregido**: ahorro modesto adicional en encoder para audio corto.
- Las estimaciones del plan anterior (-50%, -55%) eran optimistas y no fundamentadas.
